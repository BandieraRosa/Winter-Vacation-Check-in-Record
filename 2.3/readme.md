# 2.3

从去年十一月初，我便开始研发针对RoboMaster比赛的能量机关识别与追踪算法。但是断断续续搞到了十二月底，却仍然没什么头绪，于是花了两个星期将华南理工大学华南虎战队的能量机关识别算法开源给clone了下来，试图将其融入我们目前的上位机系统中。

但这个过程比我想象中要困难。

首先便是生态与依赖的问题。华南虎开源基于纯原生c++、opencv、eigen，而我们的系统目前是基于ROS2组织起来的，双方在工具链、编译依赖等等都有着天差地别的实现。华南虎开源几乎自己将轮子都造了一遍，包括但不限于概念类型约束、坐标系管理、参数管理、可断点的可视化调试工具等等，而我们使用的几乎都是基于ROS2生态的实现。

其次便是该开源的传统视觉水平太高，我很难在不做大的改动的情况下顺利开展工作。

基于此，我在度过期末的这段时间后，做出了决定：不重复实现，只将其核心算法作为内核，以ROS2功能包包装一层即可。先达到可用状态，其余的可以先后续优化。

做出这个决定，也是受制于进度问题。五月下旬即是RoboMaster超级对抗赛，如今已经二月初，时间上早已经非常紧张，况且还需要准备三月下旬的RoboMaster高校联盟赛，若不以实用为先，恐怕很难完成这项任务。

基于此，我决定先实现将其编译为外部库。
